# Contributing to bic-spark

Thank you for your interest in contributing! Please follow these guidelines.

> [!NOTE]
> Please open an issue first. Describe the bug or feature so it can be reviewed and discussed before any work begins. 

Refer to the main readme on how to run.

## Development Rules

- Write code in **TypeScript**
- Follow existing code style and naming conventions
- Run tests before submitting: `npm test`
- Lint your code: `npm run lint`

## Submitting Changes

1. **Open an issue first** — describe the bug or feature so it can be reviewed and discussed before any work begins.
2. Wait for approval/feedback on the issue before starting work.
3. Commit your changes with a clear message (e.g. `feat: add new spark option`)
4. Push to your fork: `git push origin feat/your-feature-name`
5. Open a Pull Request against the `main` branch, referencing the issue (e.g. `Closes #42`)
6. Fill out the pull request template

## Commit Message Format

Use [Conventional Commits](https://www.conventionalcommits.org/):

- `feat:` – new feature
- `fix:` – bug fix
- `docs:` – documentation only
- `chore:` – maintenance tasks
- `test:` – adding or updating tests

## Reporting Issues

Use the GitHub issue templates for bugs or feature requests.
